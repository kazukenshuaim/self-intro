## 2. 機能要件

### 2.1. アカウント登録機能
- ユーザー名、メールアドレス、パスワードを入力することでアカウント登録できる

### 2.2. ログイン、ログアウト機能
- すでに登録されたアカウントにログインすることで、以下の機能を使用できる
- ログアウトすることで、以下の機能を使用できない状態にできる

### 2.3. 新規登録機能
- 名前 / 誕生日 / 趣味・特技を入力して登録できる

### 2.4. 一覧表示機能
- 登録済み自己紹介をすべて一覧表示できる
- 表示項目：名前、作成日時
- 登録日時の降順で表示する
- 各自己紹介から詳細画面に遷移できる

### 2.5. 詳細表示機能
- 自己紹介IDを指定して1件の詳細を表示する
- 表示項目：名前 / 誕生日 / 趣味・特技 / 作成日時

### 2.6. 自己紹介編集、削除機能
- 指定の自己紹介の名前 / 誕生日 / 趣味・特技を編集することができる
- 指定の自己紹介を削除できる

## 3. 非機能要件

### 3.1. 信頼性
- 登録済みデータが空の場合、一覧画面には登録済み自己紹介がないことが記載される

### 3.2. 保守性
- フロントエンドとバックエンドはファイルを分割して管理する

## 4. 画面一覧
| 画面名 | 説明 |
| --- | --- |
| アカウント登録画面 | アカウント登録を行う |
| ログイン画面 | ログインを行う |
| 新規登録画面 | 自己紹介を入力し登録する |
| 一覧画面 | 登録済み自己紹介を一覧表示する |
| 詳細画面 | 特定の自己紹介の詳細を表示、編集や削除をする |
| 自己紹介編集画面 | 特定の自己紹介を編集する |
| 削除確認画面 | 特定の自己紹介を本当に削除するか聞く |

## 5. API一覧

| メソッド | パス | 機能概要 | 処理内容 / 備考 |
| --- | --- | --- | --- |
| **POST** | `/accounts/register` | アカウントの登録 |
| **POST** | `/accounts/login` | ログイン |
| **POST** | `/accounts/logout` | ログアウト |
| **POST** | `/tasks/create` | 自己紹介の登録 |
| **GET** | `/tasks` | 自己紹介一覧の取得 |
| **GET** | `/tasks/{id}` | 指定IDの自己紹介詳細の取得 |
| **POST** | `/tasks/{id}/update` | 指定IDの自己紹介の編集 |
| **DELETE** | `/tasks/{id}/delete` | 指定IDの自己紹介の削除 |

## 6. データベース
データベースとしてSQLを用いる。
